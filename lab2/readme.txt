UUID (Universally Unique Identifier) – это уникальный идентификатор длиной 128 бит, который гарантирует глобальную уникальность объектов. Используется для однозначной идентификации пользователя или счета, даже если база данных распределена или работает несколько приложений.

Архитектура рабочая, не менял

S (Single Responsibility) – AccountService работает только с аккаунтами, UserService только с пользователями.
O (Open/Closed) – добавление новых типов транзакций не требует изменения существующих классов.
L (Liskov Substitution) – любые стратегии транзакций заменяемы интерфейсом TransactionStrategy.
I (Interface Segregation) – интерфейсы разделены по типам операций, нет лишних методов.
D (Dependency Inversion) – AsyncTransactionProcessor зависит от абстракций TransactionStrategy, а не конкретных классов.

Паттерны:
- Singleton – Database обеспечивает единое соединение с базой данных.
- Strategy – стратегии выполнения транзакций (Deposit, Withdraw, Freeze).
- Factory – TransactionStrategyFactory создает нужную стратегию транзакции.
- Command – Transaction используется как команда для асинхронного выполнения.
- Observer – EventBus уведомляет об окончании транзакций.
- Visitor – может быть использован для сбора статистики по транзакциям.

Асинхронность и атомарность
- AsyncTransactionProcessor использует CompletableFuture для асинхронного выполнения.
- Каждая транзакция выполняется атомарно с учетом блокировок.
- Применяемые паттерны: Command, Observer, Strategy, Singleton.

UI
- Отображение пользователей и счетов.
- Кнопки: добавить/удалить пользователя, добавить/удалить счет, пополнить, снять, перевести, заморозить/разморозить, для последних операций надо в поле для суммы её написать
- Проверка корректности ввода: сумма операций только положительная, UUID для перевода корректный.
